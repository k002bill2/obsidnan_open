Update all active dev docs with current progress:

1. **In context.md**:
   - Update "Last Updated" timestamp to now
   - Add any new decisions made during this session
   - Update "Current Issues" section with new issues found
   - Revise "Next Steps" based on actual progress
   - Add any new files that were created or modified

2. **In tasks.md**:
   - Mark completed items with [x]
   - Add any new tasks discovered during implementation
   - Update completion counts (e.g., "8/12 complete")
   - Reorder by priority if needed
   - Note any blocked items

3. **Add a session summary** at the end of context.md:
   ```markdown
   ## Session Summary - [날짜 시간]
   
   ### Accomplished
   - 완료한 작업들
   
   ### Issues Encountered
   - 발견한 문제들
   
   ### Next Priority
   - 다음 우선순위 작업
   ```

Keep updates:
- Concise but comprehensive
- Focused on actionable information
- Easy to resume from in next session
